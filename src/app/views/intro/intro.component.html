  7. Theme the list.
  8. Filtering. Should we do buttons or dropdowns? Depends on how many options there are.

Performance

It takes quite long time to render the markup for the +800 photos that was taken on sol 1000 by Curiosity. Initially I wanted to use the new Angular 7 virtual scroll but went with traditional paginaton instead to facilitate bootstrap features.

Findings

As described in the task defintion we set sol (Martian rotations) to 1000 for each query.

Unfortunally Opportunity and Spirit took few photos on their 1000 sol. It is possible to implement a date picker and provide earth_date, but I believe it's out of scope.

The API does not provide information on how many photos there is in a resultset when pagination and camera filtering is used. This bug is not tackled
